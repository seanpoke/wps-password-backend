### 1. User login

**接口地址**: `/account/login`

**请求方式**: `POST`

**请求参数**:

| 参数名   | 类型   | 必填 | 说明     |
| -------- | ------ | ---- | -------- |
| account  | String | 是   | 用户账号 |
| password | String | 是   | 用户密码 |

**请求示例**:

```json
{
  "account": "user1",
  "password": "password123"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "account": "user1",
    "name": "张三",
    "role": "user"
  }
}
```

**响应字段说明**:

| 字段    | 类型   | 说明                                               |
| ------- | ------ | -------------------------------------------------- |
| token   | String | 访问令牌，有效期 72 小时，后续请求需在请求头中携带 |
| account | String | 用户账号                                           |
| name    | String | 用户姓名                                           |
| role    | String | 用户角色 (admin 超级管理员 / user 普通用户)，默认 user  |

**错误码**:

| 状态码 | 说明                                                  |
| ------ | ----------------------------------------------------- |
| 400    | 参数错误：账号和密码不能为空 / 不能使用管理员账号登录 |
| 401    | 认证失败：账号或密码错误                              |

### 2. 刷新token（心跳保活）

**接口地址**: `/account/refresh-token`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "account": "user1",
    "name": "张三",
    "role": "user"
  }
}
```

**响应字段说明**:

| 字段    | 类型   | 说明                                               |
| ------- | ------ | -------------------------------------------------- |
| token   | String | 当前使用的访问令牌（与原token相同）                |
| account | String | 用户账号                                           |
| name    | String | 用户姓名                                           |
| role    | String | 用户角色 (admin 超级管理员 / user 普通用户)，默认 user  |

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- TokenInterceptor会自动将token的过期时间刷新为72小时
- 前端可以定期调用此接口来保持用户登录状态，避免token过期
- 建议每隔30-60分钟调用一次此接口进行心跳保活

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 401    | 未授权：token无效或已过期 |
| 500    | token刷新失败：系统异常   |

### 3. 用户登出

**接口地址**: `/account/logout`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "登出成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 从 Redis 中删除该 token，使 token 立即失效
- 登出后，该 token 无法再用于其他请求
- 前端应在用户点击登出按钮时调用此接口，并清除本地存储的 token

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 500    | 登出失败：系统异常        |

### 4. 获取文档所属人

**接口地址**: `/doc/owner`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名   | 类型   | 必填 | 说明         |
| -------- | ------ | ---- | ------------ |
| docId    | String | 是   | 文档唯一标识 |
| fileName | String | 否   | 文件名       |

**请求示例**:

```json
{
  "docId": "doc123456",
  "fileName": "test.docx"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
     "ownerAccount": "user1",
     "ownerName": "张三",
     "readAuth": true,
     "writeAuth": true
  }
}
```

**响应字段说明**:

| 字段           | 类型    | 说明                     |
|--------------| ------- | ------------------------ |
| ownerAccount | String  | 文档所属账号             |
| ownerName    | String  | 文档所属名称             |
| readAuth     | Boolean | 当前用户是否有读权限     |
| writeAuth    | Boolean | 当前用户是否有写权限     |

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 获取当前文档的所属人信息
- 如果文档不存在，则自动创建文档记录，并保存 fileName（如果提供）
- 如果文档已存在但 fileName 为空，且请求中提供了 fileName，则更新 fileName
- fileName 仅在首次创建或为空时才会保存，后续不会覆盖
- 权限判断逻辑：
  - **文档所有者**：`readAuth = true`, `writeAuth = true`（同时拥有读写权限）
  - **被授权用户/部门**：`readAuth = true`, `writeAuth = false`（只有读权限）
  - **无权限用户**：`readAuth = false`, `writeAuth = false`（无任何权限）
- 前端根据 `readAuth` 和 `writeAuth` 的值决定展示哪些操作按钮

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 400    | 参数错误：文档id不能为空 |
| 401    | 未授权                   |

### 5. 获取文档密码

**接口地址**: `/doc/password`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名        | 类型    | 必填 | 说明                                            |
| ------------- | ------- | ---- | ----------------------------------------------- |
| docId         | String  | 是   | 文档唯一标识                                    |
| encryPassword | String  | 是   | 使用公钥加密后的密码                            |
| keyVersion    | String  | 否   | 密钥版本，默认"default"                         |
| isTemp        | Boolean | 否   | 是否为临时操作，true则跳过文件存在性和所有者校验，默认false |

**请求示例**:

```json
{
  "docId": "doc123456",
  "encryPassword": "encrypted_password_string",
  "keyVersion": "default",
  "isTemp": false
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
     "password": "document_password_123"
  }
}
```

**响应字段说明**:

| 字段     | 类型   | 说明                 |
| -------- | ------ | -------------------- |
| password | String | 解密后的文档密码明文 |

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 系统会校验当前登录用户是否有该文档的访问权限
- 只有被授权的用户才能获取文档密码
- 根据 `keyVersion` 参数查询对应版本的私钥进行解密
- 如果不传 `keyVersion`，默认使用 "default" 版本
- 前端应先调用 `/config/latest-key` 接口获取最新公钥和密钥版本，用于加密密码
- **临时模式**：当 `isTemp` 为 `true` 时，系统将跳过以下校验：
  - 不检查文件是否存在于数据库
  - 不检查当前用户是否为文件所有者
  - 不检查用户是否有访问权限
  - 直接进行密码解密操作
- **正常模式**：当 `isTemp` 为 `false` 或 `null` 时，执行完整的校验流程

**错误码**:

| 状态码 | 说明                                            |
| ------ | ----------------------------------------------- |
| 400    | 参数错误：docId和encryPassword不能为空          |
| 401    | 未授权或无访问权限                              |
| 409    | 文件不存在                                      |
| 500    | 解密失败                                        |

### 7. 更新文档权限

**接口地址**: `/doc/auth/update`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名        | 类型          | 必填 | 说明                                            |
| ------------- | ------------- | ---- | ----------------------------------------------- |
| docId         | String        | 是   | 文档唯一标识                                    |
| accountDnList | Array[String] | 否   | 授权的用户账号 DN 列表                          |
| deptDnList    | Array[String] | 否   | 授权的部门 DN 列表                              |
| isTemp        | Boolean       | 否   | 是否为临时操作，true则跳过文件存在性和所有者校验，默认false |

**请求示例**:

```json
{
  "docId": "doc123456",
  "accountDnList": [
    "cn=user1,ou=开发组,ou=技术部,dc=example,dc=com",
    "cn=user2,ou=市场部,dc=example,dc=com"
  ],
  "deptDnList": [
    "ou=开发组,ou=技术部,dc=example,dc=com"
  ],
  "isTemp": false
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "操作成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- **仅文档所有者**可以更新授权关系
- 更新时会先删除该文档的所有旧授权记录，然后插入新的授权记录
- accountDnList 和 deptDnList 可以同时为空，表示取消所有授权
- **临时模式**：当 `isTemp` 为 `true` 时，系统将跳过以下校验：
  - 不检查文件是否存在于数据库
  - 不检查当前用户是否为文件所有者
  - 直接执行授权更新操作
- **正常模式**：当 `isTemp` 为 `false` 或 `null` 时，执行完整的校验流程（仅文档所有者可操作）

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 400    | 参数错误：docId不能为空  |
| 401    | 未授权                   |
| 409    | 文件不存在或无操作权限   |
| 500    | 系统异常                 |

### 8. 上报保存记录

**接口地址**: `/doc/save/log`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名          | 类型          | 必填 | 说明                   |
| --------------- | ------------- | ---- | ---------------------- |
| docId           | String        | 是   | 文档唯一标识           |
| path            | String        | 是   | 文件完整路径           |
| keyVersion      | String        | 是   | 密钥版本号             |
| beforePassword  | String        | 否   | 之前的密码（ECC加密）  |
| afterPassword   | String        | 否   | 当前的密码（ECC加密）  |
| possiblePassword | Array[String] | 否   | 可能的密码集合（ECC加密）|
| platform        | String        | 否   | 操作来源平台（win/android） |

**请求示例**:

```json
{
  "docId": "doc123456",
  "path": "c://desktop//a.docx",
  "keyVersion": "default",
  "beforePassword": "encrypted_before_password",
  "afterPassword": "encrypted_after_password",
  "possiblePassword": ["encrypted_pwd1", "encrypted_pwd2", "encrypted_pwd3"],
  "platform": "win"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "操作成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 根据 `keyVersion` 参数查询对应的私钥配置
- 使用 ECC 私钥对 `beforePassword`、`afterPassword`、`possiblePassword` 进行解密
- 记录用户点击保存文档时，插件此时的密码元数据信息（解密后的明文）
- 每次插件写密码都会上报这些元数据信息
- 方便后续找回密码和溯源
- `platform` 字段用于标识操作来源平台，可选值为 "win" 或 "android"
- **重要**：所有密码字段必须使用 ECC 公钥加密后传输，确保安全性

**错误码**:

| 状态码 | 说明                                      |
| ------ | ----------------------------------------- |
| 400    | 参数错误：docId、path和keyVersion不能为空 |
| 500    | beforePassword解密失败                    |
| 500    | afterPassword解密失败                     |
| 500    | possiblePasswordList第N个密码解密失败     |
| 500    | 未找到对应的配置密钥                      |
| 500    | 系统异常                                  |

### 9. 获取LDAP配置

**接口地址**: `/config/ldap`

**请求方式**: `GET`

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "url": "ldap://192.168.1.100:389",
    "base": "DC=greenet,DC=com,DC=cn",
    "username": "admin@greenet.com.cn",
    "trees": [
      "OU=武汉绿网,OU=绿色网络,DC=greenet,DC=com,DC=cn"
    ]
  }
}
```

**响应字段说明**:

| 字段     | 类型         | 说明                  |
| -------- | ------------ | --------------------- |
| url      | String       | LDAP服务器地址        |
| base     | String       | LDAP基础DN            |
| username | String       | LDAP管理员账号        |
| trees    | Array[String]| LDAP组织树根节点列表  |

### 12. 公钥加密

**接口地址**: `/config/encrypt`

**请求方式**: `POST`

**Content-Type**: `application/json`

**请求参数**:

| 参数名     | 类型   | 必填 | 说明                                       |
| ---------- | ------ | ---- | ------------------------------------------ |
| text       | String | 是   | 待加密的原始字符串                         |
| keyVersion | String | 否   | 密钥版本，默认"default"，可使用"latest"获取最新密钥 |

**请求示例**:

```json
{
  "text": "test123",
  "keyVersion": "default"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "encryptedText": "base64_encoded_encrypted_string",
    "originalLength": "7",
    "encryptedLength": "128",
    "keyVersion": "default"
  }
}
```

**响应字段说明**:

| 字段            | 类型   | 说明                 |
| --------------- | ------ | -------------------- |
| encryptedText   | String | ECC加密后的密文      |
| originalLength  | String | 原始明文长度         |
| encryptedLength | String | 加密后密文长度       |
| keyVersion      | String | 使用的密钥版本       |

**业务说明**:

- 使用系统公钥对传入的字符串进行ECC加密（ECIES方案）
- 主要用于前端加密密码后传给后端
- 可通过 `keyVersion` 参数指定使用哪个版本的公钥
- 如果传入 "latest"，将使用优先级最高的公钥（order_num最大）
- 如果不传 `keyVersion`，默认使用 "default" 版本
- 无需认证即可访问

**使用示例**:

```bash
# 使用默认密钥版本
curl -X POST "http://localhost:8080/config/encrypt" \
  -H "Content-Type: application/json" \
  -d '{"text":"test123","keyVersion":"default"}'

# 使用指定密钥版本
curl -X POST "http://localhost:8080/config/encrypt" \
  -H "Content-Type: application/json" \
  -d '{"text":"test123","keyVersion":"v2"}'

# 使用最新密钥版本（推荐）
curl -X POST "http://localhost:8080/config/encrypt" \
  -H "Content-Type: application/json" \
  -d '{"text":"test123","keyVersion":"latest"}'
```

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 400    | 参数错误：text不能为空   |
| 500    | 加密失败：系统异常       |

### 14. 获取最新密钥信息

**接口地址**: `/config/latest-key`

**请求方式**: `GET`

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "keyVersion": "default",
    "publicKey": "MFkwEwYHKoZIzj0CAQYIKoZIzj0DAQcDQgAEuY2/Hz7c7gM0O8P/8VYjDasWhdW4jyS99+Xwyghe+CVFko7KPeamzaOsUffIHQz0VAA8RH9MV1BYyuZAJ7X05Q=="
  }
}
```

**响应字段说明**:

| 字段       | 类型   | 说明                       |
| ---------- | ------ | -------------------------- |
| keyVersion | String | 当前优先级最高的密钥版本   |
| publicKey  | String | ECC公钥（Base64编码）      |

**业务说明**:

- 获取系统当前优先级最高的密钥版本和公钥
- 从 `config_secret_key` 表中查询 `order_num` 最大的记录
- `order_num` 越大，优先级越高
- 前端应在初始化时调用此接口获取公钥，用于加密密码
- 建议缓存该信息，避免频繁请求
- 当需要密钥轮换时，插入新密钥并设置更大的 `order_num`，客户端再次调用此接口即可获取新密钥

**使用场景**:

1. **客户端初始化**: 启动时获取公钥用于加密数据
2. **密钥轮换**: 定期调用此接口检查是否有新密钥
3. **多版本管理**: 根据 `keyVersion` 选择合适的密钥进行加解密

**错误码**:

| 状态码 | 说明                         |
| ------ | ---------------------------- |
| 500    | 系统配置错误：未找到配置密钥 |
