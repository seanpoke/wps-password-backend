### 1. User login

**接口地址**: `/account/login`

**请求方式**: `POST`

**请求参数**:

| 参数名   | 类型   | 必填 | 说明     |
| -------- | ------ | ---- | -------- |
| account  | String | 是   | 用户账号 |
| password | String | 是   | 用户密码 |

**请求示例**:

```json
{
  "account": "user1",
  "password": "password123"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "account": "user1",
    "name": "张三"
  }
}
```

**响应字段说明**:

| 字段    | 类型   | 说明                                               |
| ------- | ------ | -------------------------------------------------- |
| token   | String | 访问令牌，有效期 72 小时，后续请求需在请求头中携带 |
| account | String | 用户账号                                           |
| name    | String | 用户姓名                                           |

**错误码**:

| 状态码 | 说明                                                  |
| ------ | ----------------------------------------------------- |
| 400    | 参数错误：账号和密码不能为空 / 不能使用管理员账号登录 |
| 401    | 认证失败：账号或密码错误                              |

### 2. 刷新token（心跳保活）

**接口地址**: `/account/refresh-token`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
    "token": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "account": "user1",
    "name": "张三"
  }
}
```

**响应字段说明**:

| 字段    | 类型   | 说明                                               |
| ------- | ------ | -------------------------------------------------- |
| token   | String | 当前使用的访问令牌（与原token相同）                |
| account | String | 用户账号                                           |
| name    | String | 用户姓名                                           |

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- TokenInterceptor会自动将token的过期时间刷新为72小时
- 前端可以定期调用此接口来保持用户登录状态，避免token过期
- 建议每隔30-60分钟调用一次此接口进行心跳保活

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 401    | 未授权：token无效或已过期 |
| 500    | token刷新失败：系统异常   |

### 3. 用户登出

**接口地址**: `/account/logout`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**: 无

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "登出成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 从 Redis 中删除该 token，使 token 立即失效
- 登出后，该 token 无法再用于其他请求
- 前端应在用户点击登出按钮时调用此接口，并清除本地存储的 token

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 500    | 登出失败：系统异常        |

### 5. 获取文档密码

**接口地址**: `/doc/password`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名        | 类型   | 必填 | 说明                 |
| ------------- | ------ | ---- | -------------------- |
| docId         | String | 是   | 文档唯一标识         |
| encryPassword | String | 是   | 使用公钥加密后的密码 |

**请求示例**:

```json
{
  "docId": "doc123456",
  "encryPassword": "encrypted_password_string"
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": {
     "password": "document_password_123"
  }
}
```

**响应字段说明**:

| 字段     | 类型   | 说明                 |
| -------- | ------ | -------------------- |
| password | String | 解密后的文档密码明文 |

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 系统会校验当前登录用户是否有该文档的访问权限
- 只有被授权的用户才能获取文档密码

**错误码**:

| 状态码 | 说明                                            |
| ------ | ----------------------------------------------- |
| 400    | 参数错误：docId和encryPassword不能为空          |
| 401    | 未授权或无访问权限                              |
| 409    | 文件不存在                                      |
| 500    | 解密失败                                        |

### 7. 更新文档权限

**接口地址**: `/doc/auth/update`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名        | 类型          | 必填 | 说明                         |
| ------------- | ------------- | ---- | ---------------------------- |
| docId         | String        | 是   | 文档唯一标识                 |
| accountDnList | Array[String] | 否   | 授权的用户账号 DN 列表       |
| deptDnList    | Array[String] | 否   | 授权的部门 DN 列表           |

**请求示例**:

```json
{
  "docId": "doc123456",
  "accountDnList": [
    "cn=user1,ou=开发组,ou=技术部,dc=example,dc=com",
    "cn=user2,ou=市场部,dc=example,dc=com"
  ],
  "deptDnList": [
    "ou=开发组,ou=技术部,dc=example,dc=com"
  ]
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "操作成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- **仅文档所有者**可以更新授权关系
- 更新时会先删除该文档的所有旧授权记录，然后插入新的授权记录
- accountDnList 和 deptDnList 可以同时为空，表示取消所有授权

**错误码**:

| 状态码 | 说明                     |
| ------ | ------------------------ |
| 400    | 参数错误：docId不能为空  |
| 401    | 未授权                   |
| 409    | 文件不存在或无操作权限   |
| 500    | 系统异常                 |

### 8. 上报保存记录

**接口地址**: `/doc/save/log`

**请求方式**: `POST`

**请求头**:

| 参数名 | 类型   | 必填 | 说明         |
| ------ | ------ | ---- | ------------ |
| token  | String | 是   | 登录返回的令牌 |

**请求参数**:

| 参数名          | 类型          | 必填 | 说明           |
| --------------- | ------------- | ---- | -------------- |
| docId           | String        | 是   | 文档唯一标识   |
| path            | String        | 是   | 文件完整路径   |
| beforePassword  | String        | 否   | 之前的密码     |
| afterPassword   | String        | 否   | 当前的密码     |
| possiblePasword | Array[String] | 否   | 可能的密码集合 |

**请求示例**:

```json
{
  "docId": "doc123456",
  "path": "c://desktop//a.docx",
  "beforePassword": "abc",
  "afterPassword": "123",
  "possiblePasword": ["55","34","abc","123"]
}
```

**响应示例**:

```json
{
  "message": "操作成功",
  "status": 200,
  "data": "操作成功"
}
```

**业务说明**:

- 系统会从Token中获取当前登录用户信息
- 记录用户点击保存文档时，插件此时的密码元数据信息
- 每次插件写密码都会上报这些元数据信息
- 方便后续找回密码和溯源

**错误码**:

| 状态码 | 说明                          |
| ------ | ----------------------------- |
| 400    | 参数错误：docId和path不能为空 |
| 500    | 系统异常                      |
