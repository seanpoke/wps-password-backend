# 文档管理系统技术文档

## 1. 项目概述

本项目是基于 SpringBoot3 + LDAP + MySQL8 + Redis 技术栈开发的文档管理系统后端接口系统，主要功能包括用户认证、文件信息管理、密码解密、LDAP 结构查询及文档授权管理。

## 2. 技术栈

| 技术组件 | 版本 | 核心用途 |
|---------|------|----------|
| SpringBoot | 3.2.0 | 后端框架，快速构建 REST 接口 |
| MySQL | 8.x | 存储文件信息、授权关系、操作日志 |
| Redis | - | 缓存 token、LDAP 结构数据，设置过期时间 |
| LDAP | - | 用户身份认证（用户名密码校验） |
| JWT | 0.9.1 | 生成和管理访问令牌 |
| RSA | - | 非对称加密算法，用于密码加密和解密 |

## 4. 核心功能模块

### 4.1 用户认证模块
- 实现了基于 LDAP 的用户身份认证
- 生成唯一 token 并存储到 Redis，设置 72 小时过期
- 实现了全局 token 拦截器，自动校验和刷新 token

### 4.2 文件信息管理模块
- 实现了文件信息的查询和初始化
- 生成 RSA 公私钥对并存储到数据库
- 支持文件所有权管理

### 4.3 密码解密模块
- 实现了基于 RSA 私钥的密码解密
- 实现了访问权限校验
- 记录操作日志

### 4.4 LDAP 结构查询模块
- 实现了 LDAP 目录结构的查询
- 使用 Redis 缓存结构数据，设置 24 小时过期
- 对查询结果进行排序

### 4.5 文档授权管理模块
- 实现了文档授权关系的全量更新
- 支持用户和部门级别的授权
- 使用事务控制保证数据一致性

## 5. API 接口详情

### 5.1 登录接口
- **接口地址**: `/account/login`
- **请求方式**: POST
- **入参**:
  | 字段名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | account | String | 是 | 用户名 |
  | password | String | 是 | 密码 |
- **出参**:
  | 字段名 | 类型 | 说明 |
  |-------|------|------|
  | token | String | 访问令牌 |
- **核心逻辑**: LDAP 认证 -> 生成 token -> 存储到 Redis

### 5.2 获取文件信息接口
- **接口地址**: `/doc/info?docId={docId}`
- **请求方式**: GET
- **入参**:
  | 字段名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | docId | String | 是 | 文件唯一标识 |
- **出参**:
  | 字段名 | 类型 | 说明 |
  |-------|------|------|
  | publicKey | String | 公钥 |
  | owner | String | 文档所属人账号 |
  | uid | String | 文件唯一标识 |
- **核心逻辑**: 查询文件信息 -> 不存在则生成公私钥对 -> 存储到数据库

### 5.3 查询文档密码接口
- **接口地址**: `/doc/password`
- **请求方式**: POST
- **入参**:
  | 字段名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | docId | String | 是 | 文件唯一标识 |
  | account | String | 是 | 用户账号 |
  | encryPassword | String | 是 | 加密后的密码 |
- **出参**:
  | 字段名 | 类型 | 说明 |
  |-------|------|------|
  | password | String | 解密后的密码 |
- **核心逻辑**: 权限校验 -> 使用私钥解密 -> 记录操作日志

### 5.4 查询 LDAP 结构接口
- **接口地址**: `/doc/ldap/info`
- **请求方式**: GET
- **出参**:
  | 字段名 | 类型 | 说明 |
  |-------|------|------|
  | name | String | 部门/节点名称 |
  | fullPath | String | LDAP 完整路径 |
  | accountList | List<String> | 当前部门下的账号集合 |
  | deptList | List<Item> | 子部门集合 |
- **核心逻辑**: 查询缓存 -> 缓存不存在则查询 LDAP -> 排序 -> 存储到缓存

### 5.5 更新文档授权接口
- **接口地址**: `/doc/updateRel`
- **请求方式**: POST
- **入参**:
  | 字段名 | 类型 | 必填 | 说明 |
  |-------|------|------|------|
  | docId | String | 是 | 文件唯一标识 |
  | operate | String | 是 | 操作人账号 |
  | accountFullList | List<String> | 否 | 被授权用户账号集合 |
  | deptFullList | List<String> | 否 | 被授权部门集合 |
- **核心逻辑**: 权限校验 -> 删除旧授权 -> 插入新授权 -> 事务提交

## 7. 核心技术实现

### 7.1 Token 校验与刷新
- 实现了 `TokenInterceptor` 拦截器，对非登录接口进行拦截
- 从 Redis 查询 token 是否存在，存在则重置过期时间为 72 小时
- 不存在则返回 401 未授权

### 7.2 LDAP 用户校验
- 集成了 Spring LDAP 模板，实现了用户身份校验
- 提供了用户和部门的 LDAP 路径查询功能

### 7.3 Redis 缓存策略
- Token 缓存：key=token，value=account，过期时间 72 小时
- LDAP 结构缓存：key=LDAP_STRUCTURE，value=序列化后的结构数据，过期时间 24 小时

### 7.4 RSA 加解密
- 使用 RSA 非对称加密算法生成公私钥对
- 解密密码时使用私钥，保证数据安全性

### 7.5 事务控制
- 在更新文档授权的方法上添加 `@Transactional` 注解
- 保证删除旧数据、插入新数据的原子性

## 8. 异常处理

- **参数异常**：入参非空校验失败时，返回 400 Bad Request，提示具体缺失字段
- **授权异常**：token 无效 / 无操作权限时，返回 401 Unauthorized
- **业务异常**：文件不存在、LDAP 查询失败等，返回 409 Conflict，提示具体业务原因
- **系统异常**：数据库 / Redis/LDAP 连接失败等，返回 500 Internal Server Error，记录详细日志

## 10. 总结

本系统实现了文档管理系统的核心功能，包括用户认证、文件信息管理、密码解密、LDAP 结构查询和文档授权管理。系统采用了 SpringBoot 3 作为后端框架，集成了 LDAP、MySQL 和 Redis 等技术，实现了高效、安全的文档管理服务。

系统的设计遵循了 RESTful API 规范，接口清晰，逻辑严谨，具有良好的可扩展性和维护性。通过 token 认证、RSA 加密、事务控制等技术，保证了系统的安全性和数据一致性。
