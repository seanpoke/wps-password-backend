# 日志配置规范

## 配置文件位置

- **Logback配置文件**: `src/main/resources/logback-spring.xml`
- **应用配置**: `src/main/resources/application.yml`

## 日志级别说明

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| DEBUG | 调试信息，详细的系统运行细节 | 方法入参、SQL语句、变量值 |
| INFO | 重要业务流程节点 | 用户登录、操作成功、系统启动 |
| WARN | 潜在问题，但不影响系统运行 | 配置缺失使用默认值、重试操作 |
| ERROR | 错误信息，影响功能正常使用 | 异常堆栈、数据库连接失败 |

## 日志输出策略

### 1. 控制台输出 (CONSOLE)
- **格式**: `日期 [线程] 级别 类名 - 消息`
- **用途**: 开发环境实时查看日志
- **示例**: `2026-04-23 10:30:45.123 [http-nio-8081-exec-1] INFO  c.docauth.controller.AccountController - 用户登录成功`

### 2. 文件输出

#### 全量日志 (all.log)
- **文件**: `logs/wps-password-backend-all.log`
- **保留**: 30天
- **容量上限**: 10GB
- **内容**: 所有级别的日志

#### INFO日志 (info.log)
- **文件**: `logs/wps-password-backend-info.log`
- **保留**: 30天
- **容量上限**: 5GB
- **内容**: 仅INFO级别日志

#### WARN日志 (warn.log)
- **文件**: `logs/wps-password-backend-warn.log`
- **保留**: 30天
- **容量上限**: 2GB
- **内容**: 仅WARN级别日志

#### ERROR日志 (error.log)
- **文件**: `logs/wps-password-backend-error.log`
- **保留**: 90天（错误日志需要更长保留期）
- **容量上限**: 5GB
- **内容**: 仅ERROR级别日志

## 滚动策略

- **按时间滚动**: 每天生成新的日志文件
- **文件命名**: `{应用名}-{级别}.yyyy-MM-dd.log`
- **自动清理**: 超过保留天数或总容量上限的日志会被自动删除

## 框架日志级别配置

```xml
<!-- Spring框架 -->
org.springframework: INFO
org.springframework.web: INFO
org.springframework.boot: INFO

<!-- Hibernate/JPA -->
org.hibernate: WARN
org.hibernate.SQL: DEBUG          <!-- SQL语句 -->
org.hibernate.type.descriptor.sql.BasicBinder: TRACE  <!-- SQL参数 -->

<!-- MySQL驱动 -->
com.mysql: WARN

<!-- 项目业务代码 -->
com.docauth: DEBUG                <!-- 开发环境可调为DEBUG -->

<!-- Redis -->
org.springframework.data.redis: INFO
```

## 日志使用规范

### 1. Controller层
```java
@Slf4j
@RestController
public class AccountController {
    
    @PostMapping("/login")
    public ApiResponse<LoginResponse> login(@RequestBody LoginRequest request) {
        log.info("收到登录请求, username: {}", request.getUsername());
        try {
            // 业务逻辑
            log.info("用户登录成功, username: {}", username);
            return ApiResponse.success(response);
        } catch (Exception e) {
            log.error("用户登录失败, username: {}", request.getUsername(), e);
            return ApiResponse.error("登录失败");
        }
    }
}
```

### 2. Service层
```java
@Slf4j
@Service
public class DocService {
    
    public DocPasswordResponse getPassword(String docId) {
        log.debug("查询文档密码, docId: {}", docId);
        // 业务逻辑
        log.info("文档密码查询成功, docId: {}", docId);
        return response;
    }
}
```

### 3. 日志记录要点

✅ **应该记录的日志**:
- 接口请求参数（脱敏后）
- 关键业务流程节点
- 异常信息和堆栈
- 外部调用结果
- 权限验证结果

❌ **不应该记录的日志**:
- 敏感信息（密码、密钥、身份证号等）
- 大量重复的调试信息
- 无意义的日志（如 "进入方法"、"退出方法"）

### 4. 日志格式建议

```java
// ✅ 好的做法：包含上下文信息
log.info("文档密码更新成功, docId: {}, userId: {}", docId, userId);

// ❌ 不好的做法：缺少上下文
log.info("更新成功");

// ✅ 好的做法：异常时记录完整堆栈
log.error("数据库操作失败, docId: {}", docId, e);

// ❌ 不好的做法：只记录异常消息
log.error("数据库操作失败: {}", e.getMessage());
```

## 生产环境调整

在生产环境中，建议调整以下配置：

1. **修改日志级别**: 将 `com.docauth` 从 DEBUG 改为 INFO
   ```xml
   <logger name="com.docauth" level="INFO"/>
   ```

2. **关闭SQL日志**: 如果不需要SQL调试，可以关闭
   ```xml
   <logger name="org.hibernate.SQL" level="WARN"/>
   <logger name="org.hibernate.type.descriptor.sql.BasicBinder" level="WARN"/>
   ```

3. **动态调整**: 可以通过 Spring Boot Actuator 动态调整日志级别
   ```bash
   # 查看当前日志级别
   curl http://localhost:8081/actuator/loggers/com.docauth
   
   # 修改日志级别
   curl -X POST http://localhost:8081/actuator/loggers/com.docauth \
     -H "Content-Type: application/json" \
     -d '{"configuredLevel": "DEBUG"}'
   ```

## 日志文件位置

- **开发环境**: `./logs/` （项目根目录下的logs文件夹）
- **Docker环境**: `/app/logs/` （容器内路径，需挂载到宿主机）

## 监控和告警

建议对 ERROR 日志进行监控：
- 使用 ELK (Elasticsearch + Logstash + Kibana) 收集和分析日志
- 配置告警规则：当 ERROR 日志数量超过阈值时发送通知
- 定期审查 WARN 日志，发现潜在问题

## 注意事项

1. **日志文件大小**: 定期检查日志文件大小，确保滚动策略正常工作
2. **磁盘空间**: 监控磁盘使用情况，避免因日志占满磁盘导致服务不可用
3. **性能影响**: 避免在高频调用的方法中记录过多 DEBUG 日志
4. **敏感信息**: 记录日志前务必对敏感信息进行脱敏处理
5. **异步日志**: 如果日志成为性能瓶颈，可以考虑使用 AsyncAppender
