# 日志配置快速指南

## 📁 配置文件说明

| 文件 | 路径 | 用途 |
|------|------|------|
| logback-spring.xml | `src/main/resources/` | Logback日志框架主配置 |
| application.yml | `src/main/resources/` | 开发环境应用配置 |
| application-prod.yml.example | `config/` | 生产环境配置示例 |

## 🚀 快速开始

### 1. 日志文件位置

```
项目根目录/
└── logs/
    ├── wps-password-backend-all.log      # 所有日志
    ├── wps-password-backend-info.log     # INFO级别
    ├── wps-password-backend-warn.log     # WARN级别
    ├── wps-password-backend-error.log    # ERROR级别
    ├── wps-password-backend-all.2026-04-23.log    # 历史日志（按天滚动）
    └── ...
```

### 2. 代码中使用日志

```java
import lombok.extern.slf4j.Slf4j;

@Slf4j
@RestController
public class YourController {
    
    public void yourMethod() {
        // DEBUG: 调试信息
        log.debug("调试信息: {}", variable);
        
        // INFO: 业务流程节点
        log.info("用户登录成功, userId: {}", userId);
        
        // WARN: 潜在问题
        log.warn("配置项缺失，使用默认值: {}", defaultValue);
        
        // ERROR: 错误异常
        log.error("操作失败", exception);
    }
}
```

## ⚙️ 关键配置

### 开发环境（logback-spring.xml）

```xml
<!-- 业务代码日志级别 -->
<logger name="com.docauth" level="DEBUG"/>

<!-- SQL日志 -->
<logger name="org.hibernate.SQL" level="DEBUG"/>
<logger name="org.hibernate.type.descriptor.sql.BasicBinder" level="TRACE"/>
```

### 生产环境（application-prod.yml）

```yaml
logging:
  level:
    com.docauth: INFO                    # 降低日志级别
    org.hibernate.SQL: WARN              # 关闭SQL日志
    org.hibernate.type.descriptor.sql.BasicBinder: WARN
```

## 📊 日志级别选择

| 场景 | 级别 | 示例 |
|------|------|------|
| 方法入参、变量值 | DEBUG | `log.debug("查询参数: docId={}", docId)` |
| 接口请求、操作成功 | INFO | `log.info("文档密码更新成功")` |
| 配置缺失、重试 | WARN | `log.warn("Redis连接超时，尝试重连")` |
| 异常、失败 | ERROR | `log.error("数据库连接失败", e)` |

## 🔍 常见问题

### Q1: 如何查看实时日志？

```bash
# 查看所有日志
tail -f logs/wps-password-backend-all.log

# 只查看ERROR日志
tail -f logs/wps-password-backend-error.log

# 搜索特定关键词
grep "用户登录" logs/wps-password-backend-info.log
```

### Q2: Docker环境中日志在哪里？

```bash
# 进入容器
docker exec -it wps_app_container sh

# 查看日志
ls -la /app/logs/
tail -f /app/logs/wps-password-backend-error.log

# 或在宿主机查看（已挂载）
tail -f ./logs/wps-password-backend-error.log
```

### Q3: 日志文件太大怎么办？

日志会自动滚动和清理：
- 每天生成新文件
- 保留30天（ERROR保留90天）
- 超过容量上限自动删除旧日志

如需手动清理：
```bash
# 删除7天前的日志
find logs/ -name "*.log.*" -mtime +7 -delete
```

### Q4: 如何动态调整日志级别？

```bash
# 查看当前级别
curl http://localhost:8081/actuator/loggers/com.docauth

# 修改为DEBUG
curl -X POST http://localhost:8081/actuator/loggers/com.docauth \
  -H "Content-Type: application/json" \
  -d '{"configuredLevel": "DEBUG"}'

# 恢复为INFO
curl -X POST http://localhost:8081/actuator/loggers/com.docauth \
  -H "Content-Type: application/json" \
  -d '{"configuredLevel": "INFO"}'
```

## ⚠️ 注意事项

1. **敏感信息脱敏**
   ```java
   // ❌ 错误：记录密码
   log.info("用户登录, password: {}", password);
   
   // ✅ 正确：脱敏处理
   log.info("用户登录, username: {}", username);
   ```

2. **异常日志要包含堆栈**
   ```java
   // ❌ 错误
   log.error("操作失败: {}", e.getMessage());
   
   // ✅ 正确
   log.error("操作失败, docId: {}", docId, e);
   ```

3. **避免高频DEBUG日志**
   ```java
   // ❌ 错误：循环中大量DEBUG
   for (Item item : items) {
       log.debug("处理项: {}", item);  // 可能产生数千条日志
   }
   
   // ✅ 正确：只在关键点记录
   log.debug("开始处理{}个项", items.size());
   // ... 处理逻辑
   log.debug("处理完成");
   ```
